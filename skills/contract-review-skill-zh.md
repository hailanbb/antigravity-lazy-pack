# 合同初审 Skill（中文增强版）

## 使用边界
本 Skill 用于合同风险提示、结构化整理和初步审阅，不替代律师意见。任何法律结论、诉讼策略、最终签署意见，都应由专业法务或律师确认。

## 工作流程
1. **识别合同类型**：销售、采购、服务外包、软件开发、NDA、代理、租赁等。
2. **抽取基础信息**：合同主体、金额、期限、交付物、付款节点、验收条件、争议管辖。
3. **加载风险清单**：按合同类型匹配通用风险或特定类型风险（如软件外包）。
4. **输出风险表**：风险项、原文位置、风险说明、修改建议、确认人。
5. **提示人工确认**：对不确定内容标记“需人工确认”，禁止编造条款和页码。
6. **输出最终摘要**：高风险项、中风险项、建议补充条款、签署前确认清单。

## 输出格式
- 合同基本信息
- 关键条款摘要
- 风险清单表
- 建议修改项
- 需人工确认项
- 使用边界说明

## 参考资料与模板 (References & Templates)

### 1. 风险清单参考 (references/risk_checklist.md)
```markdown
# 合同风险清单

## 通用风险
- 合同主体不一致或简称不清
- 金额、税率、币种、付款节点不明确
- 验收标准缺失或仅写“验收合格”
- 违约责任无上限或明显不对等
- 交付范围、交付物格式、交付时间不明确
- 知识产权归属不清
- 保密期限、保密范围不明确
- 单方解除条件过宽
- 争议管辖地点对己方明显不利

## 软件/外包合同重点
- 源码、文档、部署权限归属
- 需求变更是否计费
- 验收失败后的整改周期
- 线上故障责任边界
- 第三方组件授权责任
```

## 脚本参考 (Scripts)

### 1. 风险表生成脚本 (scripts/risk_table_template.py)
```python
#!/usr/bin/env python3
import csv, sys
fields = ["risk_level", "risk_item", "source_location", "description", "suggestion", "owner"]
writer = csv.DictWriter(sys.stdout, fieldnames=fields)
writer.writeheader()
writer.writerow({
    "risk_level":"高",
    "risk_item":"违约责任无上限",
    "source_location":"第6页第9条",
    "description":"赔偿范围没有上限，可能导致不可控责任。",
    "suggestion":"建议设置不超过合同总金额或已付款金额的赔偿上限。",
    "owner":"法务确认"
})
```
