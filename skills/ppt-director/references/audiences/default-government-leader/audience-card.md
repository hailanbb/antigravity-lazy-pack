# Audience Card: Default Government Leader

## Profile

适用于政务汇报、数字化改革项目汇报、区域治理方案、国企技术运营汇报。默认受众是时间有限、重视政策对标和落地成效的领导或管理层。

## Attention Budget

- 总汇报通常 10-20 分钟。
- 前 3 页必须讲清楚：为什么重要、现在到哪一步、下一步要什么支持。
- 单页停留时间通常很短，标题必须直接表达结论。

## Wants

- 对齐上位政策和战略任务。
- 讲清总抓手、实施路径、时间节点。
- 有量化指标、对标案例、阶段成果。
- 风险可控，责任边界清晰。
- 能体现群众、企业、基层的获得感或实际效益。

## Dislikes

- 纯文字墙。
- 抽象概念堆叠。
- 只有系统界面，没有制度和流程变化。
- 只有愿景，没有指标、节点、责任。
- 只讲技术先进，不讲治理价值和落地效果。

## First 3 Slides Test

1. 是否 3 秒内知道主题和价值。
2. 是否 1 分钟内知道为什么现在要做。
3. 是否 3 页内知道成果、问题、下一步抓手。

## Likely Questions

- 这件事和上位政策是什么关系？
- 已经有什么阶段性成果？
- 需要领导决策什么？
- 风险点在哪里，怎么兜底？
- 这个系统或项目能不能复制推广？
- 群众、企业或基层有没有真实感知？
