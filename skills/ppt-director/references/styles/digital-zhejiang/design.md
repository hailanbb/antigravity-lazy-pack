# Design Language: digital-zhejiang

本文件是“数字浙江 PPT 模板六”的设计语言说明。它比 `style-card.md` 更完整，用于生成 PPT 前的设计对表和成稿风格检查。

来源模板：`/Users/caifeiya/Downloads/数字浙江PPT模板第三版第六套.pptx`

## 1. 风格定位

- 适用场景：政务数字化、国企技术运营、智慧城市、区域治理、数字改革、正式汇报。
- 气质关键词：正式、可信、清爽、秩序感、政务蓝白。
- 视觉重心：结构图、流程图、目录、分栏说明、圆形图标节点、平台架构。
- 生成原则：先根据内容选择最接近的页型，再让新页面靠齐该页型的视觉语法。不要机械照抄截图，也不要从空白页做成通用蓝白商务风。

## 2. 核心视觉语法

正文页采用“白底母版感”：

- 主背景：白色 `#FFFFFF`。
- 左上：浅蓝半透明几何标题底纹，承载页面标题。
- 右上：蓝色品牌角标、横条或斜切块。
- 右下：低透明浅蓝城市剪影，作为空间锚点。必须使用固定资产 `references/styles/digital-zhejiang/assets/city-skyline.png`，不得自行生成、重绘、替换为其他城市图或用 CSS/SVG 临摹。
- 局部：浅蓝半透明大三角形可用于目录、章节或强调页。

封面、章节转场、感谢页采用“深蓝几何满版”：

- 深蓝到亮蓝的几何叠层。
- 大三角形、斜切面和低透明度多边形交错。
- 中央白色大标题，视觉中心稳定。
- 品牌标识位于顶部中轴或右上区域。

## 3. 颜色系统

| 角色 | 色值 | 用法 |
| --- | --- | --- |
| 主色蓝 | `#014EC6` | 标题、编号、结构图主节点、图标圆底、重点数字、标题条、主边框 |
| 辅助橙 | `#FFAA11` | 次级模块、强调箭头、辅助节点、对比组、提示备注 |
| 亮蓝 | `#0065E2` / `#0073E2` | 细线、连接线、局部强调 |
| 正文黑 | `#000000` / `#262626` | 正文、说明文字、目录文字 |
| 浅蓝背景 | `#E7EFFD` | 信息背景、虚线框、城市剪影、弱承载区 |
| 浅灰背景 | `#F9F9F9` | 图片占位、普通内容区背景 |
| 米黄背景 | `#FFF5DC` 附近 | 提示块、备注块 |
| 白色 | `#FFFFFF` | 深蓝页文字、蓝色圆形节点内文字、卡片背景 |

比例建议：

- 正文页：白色 70%-85%，主蓝 10%-20%，橙色 3%-8%。
- 深蓝页：蓝色 90% 以上，白色文字，少量橙色或浅蓝点缀。
- 结构图：主链路用蓝色，辅助链路或审核/强调节点用橙色。

禁忌：

- 不要引入大面积紫色、红色、绿色或霓虹色。
- 不要让橙色成为主视觉。
- 不要使用花哨渐变、复杂阴影、互联网营销式 hero。

## 4. 字体系统

- 主字体：微软雅黑。
- 允许字重：Regular / Bold。
- 大标题一般使用 Bold。
- 小标题一般使用 Bold。
- 内容文字按情况使用 Regular 或 Bold。

### 4.1 参考字号

| 层级 | 推荐字号 | 字重 | 用法 |
| --- | --- | --- | --- |
| 大标题 | 36 / 44 / 60 / 72 | Bold | 封面主标题、章节标题、正文页重要观点标题 |
| 小标题 | 20 / 24 / 28 | Bold | 模块标题、流程节点标题、分栏标题、卡片标题 |
| 内容文字 | 16 / 18 | Regular 或 Bold | 正文说明、模块短句、要点文字 |
| 页脚/来源 | 12 / 14 / 16 | Regular | 来源、页码、低层级注释 |

### 4.2 典型使用

- 封面主标题：60 / 72 pt，Bold，白色；一般封面用 60，重大封面可用 72。
- 章节页标题：44 / 60 pt，Bold，白色或主蓝。
- 正文页主标题：36 / 44 pt，Bold，主蓝；普通内容页优先 36，标题短且需要强调时可用 44。
- 模块标题：20 / 24 / 28 pt，Bold；卡片、流程节点、分栏标题优先 20 或 24。
- 内容文字：16 / 18 pt，Regular 或 Bold；正文优先 18，辅助说明可 16。
- 标签/短词：16 / 18 / 20 pt，Regular 或 Bold；必须保证汇报投影可读。
- 页脚/来源：12 / 14 / 16 pt，Regular，低对比深灰或浅蓝。

避免使用微软雅黑 Light、宋体、特殊字体。

生成要求：

- 生成就绪导演稿必须明确每类文字的字体、字号和字重。
- PPTX 映射后如果字体替换导致换行、溢出、遮挡或可读性下降，判定为不通过。
- 中文清晰可读优先于局部美观；不得为了塞内容把正文压到 16 以下。

## 5. 母版固定元素

正文页尽量保留以下母版感：

- 左上标题区：x 约 60 px，y 约 20 px，标题为主蓝加粗。
- 右上品牌区：蓝色横条、角标或斜切块贴右上。
- 右下城市剪影：使用 `references/styles/digital-zhejiang/assets/city-skyline.png` 原始资产，保持透明通道和宽高比；推荐锚定右下，约从 x=760 px、y=515 px 延展到右下角，宽度约 500-620 px。不得自行生成、重绘、换图或将其做成卡通/照片风。
- 标题装饰：浅蓝倒三角或多边形底纹。

封面固定元素：

- 品牌标识在上方中轴或偏上区域。
- 主标题居中，白色加粗。
- 副标题位于主标题下方。
- 公司名或汇报单位在底部中间，低对比浅蓝或白色。

## 6. 组件语法

### 矩形

- 实心蓝矩形：关键模块、标题背景、主结构节点。
- 实心橙矩形：次级标题条、辅助模块、强调中枢。
- 空心蓝矩形：普通信息卡、内容容器。
- 空心橙矩形：次级普通信息卡。
- 浅蓝虚线矩形：背景承载、占位框、弱信息区。
- 浅灰/浅蓝底矩形：图片占位或普通内容背景。

矩形默认直角或极小圆角，不做圆角卡片堆叠。

### 圆形

- 实心蓝圆：主节点、核心角色、主要对象。
- 实心橙圆：辅助节点、次级对象。
- 空心圆/虚线圆：弱关系、背景承载、待填充状态。
- 常见结构：蓝色圆形承载白色图标，图标下方放短标签。

### 箭头

- 主链路箭头用主蓝。
- 辅助链路箭头用辅助橙。
- 可用成对箭头表达交换、协同、流转。
- 箭头保持扁平、简洁，不做 3D 或重阴影。

## 7. 页型语言

生成时不必逐页复刻截图，但要优先靠齐下列页型语言。

| 页型 | 适用内容 | 视觉骨架 |
| --- | --- | --- |
| 深蓝几何封面页 | 报告标题、项目名 | 深蓝满版几何背景，白色居中大标题，顶部品牌 |
| 白底斜切目录页 | 3-4 个章节 | 左侧“目录/CONTENT”，右侧编号目录，浅蓝斜切背景 |
| 蓝色侧栏目录页 | 强章节感报告 | 左侧深蓝侧栏，右侧目录项，右下城市剪影 |
| 三栏编号说明页 | 三个并列观点 | 中部三列，每列大编号、短标题、少量说明 |
| 纵向编号列表页 | 步骤、阶段、问题清单 | 左侧纵向编号，右侧标题+说明，条目留白 |
| 中心标题说明页 | 单一核心结论 | 中央大标题，下面一段短说明，右下城市剪影 |
| 双图说明页 | 截图、案例、对比图 | 两个浅灰图片框，下方说明，标题左上 |
| 环形五步页 | 五步流程、五类能力 | 中央圆/环，周围五个编号节点 |
| 矩阵页 | 对比、能力矩阵 | 蓝/橙标题条，浅底单元格，虚线或细边框 |
| 架构图页 | 平台、系统、数据流 | 蓝色主模块条，橙色次级中枢，浅蓝模块卡片和蓝橙连接线 |
| 圆形关系图页 | 主体生态、能力模型 | 左侧大圆核心节点，外围小圆，右侧要点 |
| Logo 墙页 | 生态伙伴、客户案例 | 多行多列 logo，白底或透明，少边框 |
| 二维码页 | 扫码入口 | 二维码居中或偏右，周围留白 |
| 深蓝几何感谢页 | 收尾 | 深蓝满版几何背景，中央“感谢/谢谢” |

## 8. 版式尺度

以 1280 x 720 px 渲染画布为参考：

- 安全边距：左右至少 60 px，上方约 22 px。
- 主内容区：x=80-1180 px，y=120-620 px。
- 左上标题位置：x 约 60 px，y 约 20 px。
- 右上品牌角标：贴右上，约 100-200 px 宽。
- 右下城市剪影：固定资产 `references/styles/digital-zhejiang/assets/city-skyline.png`，x 约 760 px 起，y 约 515 px 起，锚定右下，保持原图比例和透明背景。
- 三列布局：每列宽约 250-330 px，列间距 40-80 px。
- 标题条高度：45-70 px。

单页最多表达一个主结构或 3-5 个并列点。长解释放进演讲备注，不直接铺满页面。

## 9. 图片与截图规则

- 产品截图页可使用双图说明页或架构图页的截图承载区。
- 截图保持矩形裁切，不使用厚重阴影。
- 图片说明用 12-16 pt 深灰或黑色。
- 没有真实图片时使用浅灰占位框，并标注建议图片内容。
- 右下城市剪影不应被大图完全覆盖，除非该页明确是全图说明页。
- 右下城市剪影不是可替换插画，生成 HTML/PPTX 时必须引用固定 PNG 资产，不得自行生成。

## 10. 内容口吻

- 标题短、实、前置结论。
- 小标题控制在 4-8 个汉字。
- 正文每块 1-3 行。
- 用“平台、机制、闭环、协同、共享、治理、审核、编目”等政务数字化词汇，但避免空泛堆叠。
- 避免“极致、颠覆、赋能未来”等营销化表达。

## 11. 截图样例使用方式

`contact-sheet.png` 是设计靠齐参考，不是逐页照抄模板。

使用规则：

1. 生成前先看 `design.md`，再看 `contact-sheet.png`，最后看 `slide-type-map.md`。
2. 每页先按内容选择“最接近的页型语言”。
3. 若内容与截图页型完全匹配，则尽量靠齐其标题位置、主结构区域和组件语法。
4. 若内容需要新结构，则可以变体设计，但必须保留颜色、字体、母版固定元素、组件语法和信息密度。
5. 生成后检查：风格是否像同一套模板新增页面，而不是普通蓝白商务模板。

## 12. 生成检查清单

- 是否为 16:9 宽屏。
- 是否使用微软雅黑。
- 是否只使用主蓝、辅助橙、黑白灰和浅蓝/米黄背景。
- 正文页是否有左上标题、右上角标、右下城市剪影的母版感。
- 右下城市剪影是否引用 `city-skyline.png` 固定资产，而不是模型新画或代码重绘。
- 标题字号是否明显大于正文，且标题为蓝色加粗。
- 是否避免大段文字和营销式表达。
- 架构图是否蓝为主、橙为辅。
- 矩阵/表格标题条是否为蓝/橙实心矩形。
- 圆形图标是否为蓝底白图标。
- 封面/感谢页是否为深蓝几何背景。
- 是否避免复杂阴影、圆角卡片堆叠、花哨渐变和大面积非蓝橙配色。
