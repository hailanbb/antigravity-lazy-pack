# Standard PPT Delivery Schema

这是 PPT Director 的内容结构接口。它用于描述 PPT “讲什么、怎么组织、怎么排布”，不绑定具体设计语言。

进入代码生成时，再额外提供设计语言配置，例如 `design.md`、`slide-type-map.md` 和 contact sheet。

## Rules

- 每页标题必须是观点句，不是主题词。
- 每页要点默认 3-5 条，每条尽量不超过 20 个汉字。
- 每页只服务一个核心信息。
- 数据页标题必须说明“数据在说什么”。
- 视觉需求要明确到图表、示意图、图标、图片、二维码或纯文字。
- 页面结构导演稿必须比“视觉需求”更具体，要描述页面类型、构图、主视觉、模块关系、上屏短词、管理价值、底部总结语和通用禁忌。
- 页面结构导演稿必须写清模块间关系和面积预算：并列、支撑/底座、总分、贯穿型能力、侧挂治理层，以及主视觉/辅助内容占比。
- 页面结构导演稿中的主视觉必须能被生成模型或代码 Agent 直接转成版式，不要使用“做得高级一点”“科技感一点”这种空泛描述。
- 页面结构导演稿不得写具体设计语言、颜色、字体、母版元素或截图样例。
- 设计语言选择、页型映射和风格靠齐只在代码生成阶段执行，避免污染内容结构文档。
- 用户要求生成 PPTX 时，默认先生成 HTML 设计预览，再进入 PPTX 映射。
- HTML 生成前必须先有视觉导演优化版 MD；如果当前仍是内容型 PPT MD，需要先执行 `visual-director-optimization.md`。
- 用户提供的原始 MD 不得直接作为 HTML/PPTX 生成输入；必须先生成并使用 `页面描述_优化版MD`。
- HTML/PPTX 生成前必须先有 `生成就绪导演稿`；它要读取当前设计系统，补齐输出格式、画布参数、区域坐标、设计系统 token、字体字号参数、组件映射和生成优先级。
- `页面描述_优化版MD` 保持内容结构层，不写具体设计语言；`生成就绪导演稿` 是生成执行层，可以写来自 `design.md` 的色彩、字体、字号、字重、母版元素和组件参数。
- 固定母版资产必须在生成执行层引用原始文件路径，例如 digital-zhejiang 的 `city-skyline.png`；不得让模型自行生成、重绘或替换。
- HTML 预览必须按 `final-layout-review.md` 审查，且总分 >= 80、无 P0 问题，才允许进入 PPTX。
- PPTX 生成后必须再次按 `final-layout-review.md` 审查，且总分 >= 80、无 P0 问题，才允许输出为最终交付。
- 最终布局审查必须包含产品解决方案专家和 PPT 专家两个视角。
- PPTX 最终审查必须包含实际渲染截图，并与 HTML/PNG 预览做视觉一致性对比。
- 只检查 PPTX 可打开、页数正确、压缩包完整或文本对象存在，不足以判定通过。
- 字体字号必须进入最终审查：以实际渲染截图确认标题、小标题、正文、页脚均符合当前设计系统档位，且没有因字体替换造成换行、溢出或遮挡。
- 固定资产必须进入最终审查：确认 HTML/PPTX 渲染图中的城市剪影等母版资产来自原图，位置正确且未遮挡主视觉。
- 如果原生可编辑 PPTX 渲染后明显偏离 HTML/PNG 预览，必须修复映射；若短期无法修复且用户优先要视觉还原，可输出 `视觉一致版PPTX` 和 `可编辑原生版PPTX` 两个文件，并明确最终交付采用哪一个。
- 页面结构导演稿要控制上屏文字密度：主视觉节点内用短词，长解释放在演讲备注或交付文档正文。
- 页面文案只放真正上屏的短句；解释性背景、来源细节和完整材料放入演讲备注或素材说明。
- 页面版式和主视觉内容中的 `左侧/右侧/顶部/底部/中部/主体/侧边` 等区域词只用于排版说明，不得作为 HTML/PPTX 的可见文案前缀。
- 流程路径页中，流程主视觉必须占主体区 60% 以上；价值点不超过 3 条，每条不超过 20 字。
- 出现两个面积相当的视觉板块时，必须标记为双主视觉冲突并重排或拆页。
- 如需代码生成，必须给出图片路径或占位符路径。
