# Design Language Workflow

设计语言工作流用于防止 PPT 生成停留在“换个颜色”的层面。生成前必须先选择或确认设计语言，再用设计文档和样例图对表。

## 0. 先确认设计语言

当用户要求生成 PPT、优化 PPT 或压测 PPT 技能时，先判断用户是否指定设计语言：

- 已指定：使用该设计语言。
- 未指定但有默认值：说明将使用默认设计语言，例如 `digital-zhejiang`。
- 用户想换风格：从 `registry.yml` 列出可用设计语言，让用户选择；如果用户提供新模板，则导入为新的 style package。

不要把“默认风格”静默带入生成。必须在生成配置或设计语言对表中写明当前设计语言。

注意：设计语言信息不写进每页页面结构导演稿。页面结构稿只描述内容和排布；设计语言对表在生成阶段使用，并与页面结构稿合成为“生成就绪导演稿”。

## 1. 读取四类材料

选定设计语言后，按需读取：

- `design_doc`：完整设计语言，约束视觉 DNA、母版元素、颜色、字体、组件、页型、禁忌。
- `path`：短版 style-card，给模型或代码 Agent 的快速摘要。
- `slide_map`：内容类型到页型语言的映射。
- `assets.contact_sheet`：截图样例或缩略图，仅作为风格靠齐参考。
- `assets.city_skyline` 等固定母版资产：必须直接引用，不得由模型重新生成、重绘或替换。

`contact_sheet` 不是逐页复刻模板。它用于校准“看起来是否属于同一套设计系统”。

## 2. 生成设计语言对表

进入 HTML 设计预览或代码生成前，先输出“设计语言对表”：

```markdown
【设计语言对表】
- 当前设计语言：
- 设计系统文档：
- 样例截图/Contact Sheet：
- 核心视觉 DNA：
- 必须保留的母版感：
- 允许变体：
- 禁止偏离：

【逐页页型映射】
| 页码 | 页面内容类型 | 推荐页型语言 | 参考样例 | 必须复用的视觉语法 | 可变体空间 |
| --- | --- | --- | --- | --- | --- |
```

规则：

- “推荐页型语言”来自 `slide_map` 或 `design_doc`。
- “参考样例”可写 contact sheet 中最接近的页型名称，不要求精确页码。
- “必须复用的视觉语法”写颜色、字体字号、标题区、角标、城市剪影、蓝橙组件、直角矩形、圆形节点等。
- “固定资产”写明 registry 中的资产路径，例如 `city-skyline.png`，并标注必须引用原图。
- “可变体空间”说明哪些地方可以根据内容变化，例如流程方向、模块数量、截图位置。

## 3. 不污染页面结构导演稿

本阶段不再向每页页面结构导演稿写入设计语言锚点。

正确分层：

- 页面结构导演稿：描述页面标题、核心结论、版式关系、主视觉结构、模块内容、侧边模块、底部标签和表达重点。
- 设计语言对表：描述所选 `design.md` 如何把页面结构翻译为视觉语言。
- 生成就绪导演稿：读取页面结构导演稿和设计语言对表，补齐输出格式、画布参数、坐标区间、样式 token、组件映射、连接规则和生成优先级。
- HTML 预览 Prompt：同时读取页面结构导演稿和设计语言对表，先完成可审查的视觉排版。
- PPTX 映射 Prompt：读取已确认 HTML、页面结构导演稿和设计语言对表，完成可编辑 PPTX。

原则：

- 内容决定信息结构。
- 设计语言决定视觉语法。
- 两者在生成就绪导演稿中合成，不在内容规划阶段混写。

## 3.5 生成就绪导演稿

进入 HTML 预览或 PPTX 映射前，必须生成“生成就绪导演稿”。该稿件允许写具体设计参数，因为它属于生成执行层，不是内容结构层。

生成就绪导演稿必须包含：

- 输出格式：HTML+CSS / SVG / python-pptx / HTML+PPTX。
- 画布参数：16:9、1280×720、标题区、主体区、底部区、安全边距。
- 设计系统 token：色彩、字体、字号、字重、线条、箭头、卡片形态、母版元素。
- 固定资产引用：如右下城市剪影 PNG 的路径、锚点、尺寸范围和禁止重绘规则。
- 区域坐标或比例：主视觉区域、辅助模块区域、底部收束区。
- 页型组件映射：页面结构中的节点如何落到 `design.md` 的组件语法。
- 字体字号参数：大标题、小标题、内容文字、页脚/来源的字号档位和字重。
- 连接关系：箭头起点、终点、方向、线宽、颜色和禁止连接。
- 生成优先级：结构和关系正确优先于局部美观。

示例：

```markdown
# 生成目标
- 输出格式：HTML+CSS 预览，后续映射 PPTX
- 画布：1280×720

# 设计系统约束
- 主色：来自 design.md
- 字体字号：来自 design.md，例如 digital-zhejiang 使用微软雅黑，大标题/小标题/内容文字分档
- 固定资产：来自 registry.yml，例如 city_skyline = references/styles/digital-zhejiang/assets/city-skyline.png
- 标题区：使用当前模板的标题几何底
- 组件：直角浅蓝承载底、蓝橙标题条、编号节点

# 区域布局参数
- 主流程区域：x=120, y=140, w=730, h=420
- 侧挂治理层：x=965, y=240, w=245, h=300
- 底部价值区：x=68, y=585, w=1120, h=70

# 生成优先级
1. 主视觉和连接关系正确
2. 区域词不误上屏
3. 文案不溢出
4. 母版感一致
```

如果设计系统没有给出精确坐标，可以根据画布和页型生成合理坐标区间；但必须标明这是执行建议，并在截图审查中验证。

## 4. HTML 预览与 PPTX 映射对表要求

给编程 Agent 的 HTML 预览 Prompt 和 PPTX 映射 Prompt 必须包含：

- 当前设计语言名称。
- `design.md` 的关键约束或路径。
- `style-card.md` 摘要。
- `slide-type-map.md` 摘要。
- `contact-sheet.png` 路径或说明。
- 逐页页型映射。
- 页面结构导演稿。
- 生成就绪导演稿。
- `final-layout-review.md` 路径或审查要求。

HTML/CSS 或 PPTX 生成时优先抽象母版函数：

- `add_master_chrome`：正文页左上标题几何底、右上角标、右下城市剪影。
- `add_city_skyline_asset`：直接插入 registry 中的 `city_skyline` PNG，锚定右下，保持透明背景和宽高比，不重绘。
- `add_deep_blue_geometric_background`：封面/感谢/章节页深蓝几何背景。
- `add_blue_orange_module`：蓝橙标题条、直角模块卡片、虚线浅蓝容器。
- `add_flow_arrows`：蓝色主链路、橙色辅助链路。

## 5. 成稿风格检查与最终布局门禁

HTML 预览和 PPTX 成稿都要先做 Style Check：

```markdown
【设计语言检查】
- 设计语言：
- 是否像同一套模板新增页面：
- 颜色是否符合：
- 字体是否符合：
- 字号和字重是否符合：
- 固定资产是否引用原图而非生成重绘：
- 母版固定元素是否符合：
- 页型是否贴合内容：
- 哪些页靠齐截图：
- 哪些页是内容驱动变体：
- P0/P1/P2 修改项：
```

判断标准：

- 不要求每页版式与截图一致。
- 要求整体设计语言与 `design.md` 一致。
- 能靠齐截图页型时尽量靠齐。
- 不能为了套截图牺牲内容表达、可读性和业务逻辑。

Style Check 不能代替最终布局审查。每次 HTML 预览和 PPTX 成稿还必须按 `final-layout-review.md` 输出：

- 产品解决方案专家意见。
- PPT 专家意见。
- 100 分总分。
- P0/P1/P2 修改清单。
- 是否通过。

通过标准：总分 >= 80 且无 P0 问题。未通过不得进入下一阶段或交付最终 PPTX。
