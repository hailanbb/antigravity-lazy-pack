# PPT Director Workflow

## A. 灵感激发

适用：用户只有主题、模糊目标或零散素材。

动作：

- 明确主题、受众、目的、时长、已有素材。
- 如果已有 audience-card，直接用受众视角提问。
- 如果没有 audience-card，生成女娲受众蒸馏 Prompt。
- 输出核心信息、受众关切、前 3 页必须讲清楚的内容、2-3 种叙事结构。

## B. 内容打磨

适用：核心思路已确定，需要变成 PPT 大纲。

动作：

- 生成页数预算。
- 每页写观点型标题。
- 每页写 3-5 个要点。
- 标注图表页、图片页、结构图页、纯文字页。
- 标注需要 AI 生图的页面。
- 输出标准交付文档。
- 每页必须补充“页面结构导演稿”：页面标题、本页核心结论、页面类型、页面版式、主视觉设计、主视觉内容、模块间关系、主视觉与辅助内容面积预算、页面文案、管理价值、底部总结语、生成风险提示和出图要求。
- 页面结构导演稿要让后续代码 Agent 或设计生成模型看完即可知道这一页的信息结构、模块排布和表达重点，而不是只知道这一页讲什么。
- 页面结构导演稿必须区分“上屏短文案”和“解释性背景”，主视觉必须承担解释逻辑的作用。
- 页面结构导演稿不得写入具体设计语言、颜色、字体、母版元素或截图样例。

## B2. 页面视觉导演优化

适用：已有内容型 PPT MD、标准交付文档或第一版页面结构导演稿，准备进入 HTML 生成前。

硬规则：只要用户提供的是 PPT 内容 MD、单页描述 MD 或标准交付文档，即使内容看起来完整，也必须先执行本阶段。原始 MD 不能直接进入 HTML/PPTX。

动作：

- 读取 `references/workflows/visual-director-optimization.md`。
- 以产品解决方案专家 + PPT 视觉专家视角审查每页。
- 保留业务事实、关键素材、产品能力、指标、场景和边界，不改内容主张。
- 优化每页页面类型、版式骨架、主视觉证明对象、上屏短文案、管理价值和出图约束。
- 补充模块间关系：并列、支撑/底座、总分、贯穿型能力、侧挂治理层等。
- 补充面积预算：主视觉 >= 55%，流程路径页流程主视觉 >= 60%，辅助模块 <= 30%。
- 标记双主视觉冲突、模块无关系堆叠、贯穿型能力位置错误、箭头关系不清等生成风险。
- 把大段解释拆到主视觉内容或备注，页面文案只保留短句。
- 清洗区域词上屏风险，`左侧/右侧/顶部/底部/中部/侧边` 只作为排版指令。
- 不写具体设计语言、颜色、字体、母版元素、品牌角标或截图页型。
- 输出“页面描述_优化版 MD”和“描述优化说明”，作为 HTML 设计预览的主要输入。

## C. 视觉定义

适用：视觉导演优化版 MD 已确定，需要决定设计语言和页型。

动作：

- 先确认设计语言：用户指定则使用用户指定；未指定则说明使用 registry 默认值。
- 读取当前 `design.md`，明确视觉 DNA、母版固定元素、颜色、字体、组件语法和禁忌。
- 读取当前 style-card。
- 读取 contact sheet 或截图样例，只作为风格靠齐参考，不要求逐页复刻。
- 读取 registry 中的固定母版资产路径，例如 digital-zhejiang 的 `city_skyline`；固定资产必须直接引用，不得自行生成。
- 读取 slide-type-map，把每页内容映射到模板页型。
- 先输出“设计语言对表”：当前设计语言、设计系统文档、样例截图、核心视觉 DNA、必须保留的母版感、允许变体、禁止偏离、逐页页型映射。
- 不修改页面结构导演稿的内容。设计语言只在生成阶段作为外部约束使用。
- 根据页面结构导演稿和 slide-type-map，把每页内容映射到最接近的设计页型。
- 判断是业务 PPT 还是酷炫视觉 PPT。
- 若需要 AI 图片，生成图片 Prompt。

## D. HTML 设计预览

适用：已有标准交付文档、页面结构导演稿和设计语言说明，需要先确认最终页面效果。

动作：

- 如果输入仍是内容型 PPT MD，先返回 B2 做页面视觉导演优化。
- 如果没有 `页面描述_优化版 MD`，不得继续生成 HTML。
- 如果没有“生成就绪导演稿”，必须先读取设计语言对表、当前 `design.md` 和固定资产路径，补齐输出格式、画布参数、区域坐标/比例、样式 token、固定资产引用、组件映射、连接规则和生成优先级。
- 读取当前 toolchain。
- 读取 `references/workflows/html-preview-to-pptx-workflow.md`。
- 生成 HTML 预览、每页截图和 contact sheet。
- HTML 必须优先遵循每页的页面结构导演稿，而不是只按要点自动排版。
- HTML 必须落实模块间关系和主辅面积预算，不能把多个模块平均堆叠。
- HTML 必须读取设计语言对表，尽量靠齐 `design.md` 和样例截图的视觉语法；不要求所有版式与截图一致，但不得偏离设计系统。
- HTML 必须引用固定母版资产原图，例如 `city-skyline.png`，不得让模型生成新的城市剪影。
- 生成后必须按 `references/workflows/final-layout-review.md` 做最终布局审查。
- 审查必须包含产品解决方案专家视角和 PPT 专家视角。
- 审查必须包含模块关系审查、主视觉面积审查、页面类型专项审查。
- HTML 审查低于 80 分或存在 P0 问题，不得进入 PPTX 映射。
- HTML 审查通过后，展示 HTML 路径、contact sheet、审查摘要，请用户确认是否进入 PPTX。

## E. PPTX 映射生成

适用：HTML 预览已经通过最终布局审查，并且用户确认进入 PPTX。

动作：

- 基于已确认 HTML 预览映射成 PPTX。
- 不要默认整页截图贴入 PPT；标题、正文、矩形、箭头、流程、矩阵、分层结构应尽量保留为 PPT 原生可编辑对象。
- 复杂背景或局部插画可局部栅格化，但必须说明取舍。
- 生成后检查 PPTX 是否可打开、页数是否正确、关键文本是否可编辑、关键页面是否贴近 HTML。
- 必须导出 PPTX 实际渲染截图，并与已确认 HTML/PNG 预览并排对比。
- 再次按 `references/workflows/final-layout-review.md` 做 PPTX 最终布局审查，重点检查字体换行、箭头端点、连接关系、主辅面积、固定资产引用和区域词清洗是否在映射后漂移。
- PPTX 审查低于 80 分或存在 P0 问题，不得输出为最终交付。

## F. 迭代优化

适用：已有大纲、标准交付文档或初稿 PPT。

动作：

- Audience check：受众是否关心，前 3 页是否有效。
- Reviewer check：专家是否认可逻辑、抓手、风险闭环。
- Style check：是否符合设计语言、页型是否匹配、样例气质是否靠齐。
- Final layout check：产品解决方案专家和 PPT 专家双视角审查页面是否成立。
- 输出 `P0/P1/P2` 修改清单。

## Path Presets

### quick

输出：页数规划、每页标题、3 个要点、简版页面结构导演稿。

### controlled

输出：受众校准、大纲、逐页内容、视觉导演优化版页面结构稿、评审修改清单。

### premium

输出：受众蒸馏、完整标准交付文档、视觉导演优化版页面结构稿、设计语言对表、HTML 预览、最终布局审查、PPTX 映射、生图 Prompt、专家评审。
