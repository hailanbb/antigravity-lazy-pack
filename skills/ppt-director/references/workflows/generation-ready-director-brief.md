# Generation-Ready Director Brief Workflow

本工作流用于把“页面描述_优化版”与当前设计语言合成为“生成就绪导演稿”。它解决的问题是：页面结构稿知道“讲什么、怎么排”，但代码 Agent 或生成模型还需要知道“用什么格式输出、画布怎么切、组件怎么落、样式参数从哪里来、约束冲突时先保什么”。

## 核心分层

必须区分两层文档：

1. **内容结构层：页面描述_优化版**
   - 只写页面标题、核心结论、页面类型、主视觉、模块关系、上屏文案、管理价值和风险。
   - 不写具体设计语言名称、颜色、字体、母版元素或截图样例。
   - 作用：保证同一页内容可以换不同设计语言。

2. **生成执行层：生成就绪导演稿**
   - 读取 `design.md`、`style-card.md`、`slide-type-map.md` 和截图样例后生成。
   - 写明输出格式、画布参数、区域坐标、设计系统 token、组件映射、箭头/连接规则、优先级和审查门禁。
   - 作用：让 HTML/SVG/python-pptx/PPTX 映射具备可执行参数。

如果用户说“导演 MD 要考虑设计系统”，应生成第二层，而不是把颜色和母版元素混进第一层。

## 输入

- `页面描述_优化版.md`
- `设计语言对表`
- 当前设计语言：
  - `design.md`
  - `style-card.md`
  - `slide-type-map.md`
  - `contact-sheet.png` 或样例截图
- 输出目标：HTML+CSS / SVG / python-pptx / HTML+PPTX
- 审查规范：`final-layout-review.md`

## 输出文件建议

```text
页面描述_优化版.md              # 内容结构层
设计语言对表.md                # 设计系统选择与页型映射
生成就绪导演稿.md              # 内容结构 + 设计系统 + 执行参数
```

生成 HTML、PNG、PPTX 时，以 `生成就绪导演稿.md` 作为代码 Agent 的直接输入。

## 写作规则

- `设计系统约束` 必须来自当前 `design.md`，不能凭空编色值或字体。
- `字体与字号参数` 必须来自当前 `design.md`。digital-zhejiang 默认使用微软雅黑，且大标题、小标题、内容文字必须落在设计系统给定字号档位。
- 每个 `组件映射` 都要写明字体、字号、字重，避免 HTML/PPTX 映射时自由缩放文字。
- `固定资产引用` 必须来自 `registry.yml` 或当前 `design.md`。digital-zhejiang 的右下城市剪影必须引用 `city-skyline.png`，不得让模型自行生成。
- `区域布局参数` 可以使用 px，也可以使用百分比；复杂页面建议同时给出。
- 坐标是执行边界，不是上屏文案。
- 如果设计系统已有固定母版元素，如角标、标题底、城市剪影，应写入生成执行层。
- 如果页面结构稿中的“风格简洁、正式”等词与设计系统冲突，以设计系统为准。
- 如果用户要求换设计语言，只重做“设计语言对表”和“生成就绪导演稿”，不必重写内容结构层。
- 生成就绪导演稿必须明确输出格式，否则不进入代码生成。

## 自检清单

- 是否明确输出格式？
- 是否明确画布尺寸和主体区域？
- 是否从设计系统读取了颜色、字体、组件和母版元素？
- 是否明确字体与字号参数？
- 组件映射是否写明字体 / 字号 / 字重？
- 是否写明城市剪影资产路径并禁止自行生成？
- 是否给出关键区域坐标或比例？
- 是否给出箭头起点、终点和方向？
- 是否给出冲突优先级？
- 是否保留内容结构层的核心结论和业务事实？
- 是否没有把区域词写成上屏文案？
- 是否包含 HTML 和 PPTX 的审查门禁？
