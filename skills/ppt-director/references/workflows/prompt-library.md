## B2. 页面视觉导演优化 Prompt

```text
你是产品解决方案专家 + PPT 视觉专家。请把以下内容型 PPT MD 二次优化为“视觉导演优化版 MD”。

【输入内容】
粘贴内容型 PPT MD 或标准交付文档

【优化目标】
1. 不改业务事实，不丢关键素材。
2. 把每页改写为可生成 HTML/PPTX 的页面结构分镜。
3. 强化每页主视觉、页面类型、版式骨架、上屏短文案和管理价值。
4. 不写具体设计语言、颜色、字体、母版元素或截图样例。
5. 页面版式中的左侧/右侧/顶部/底部/中部/侧边只作为排版指令，不得作为可见模块标题。
6. 补充模块间关系和主视觉/辅助内容面积预算。
7. 标出双主视觉冲突、模块无关系堆叠、贯穿型能力位置错误、箭头关系不清等风险。

【输出要求】
- 每页按 page-structure-brief-schema.md 输出。
- 保留用户明确要求的政策、链路、产品能力、指标、场景和边界。
- 模块标题尽量 4-12 字，说明文字尽量 12-30 字。
- 主视觉内容必须承担解释逻辑，不能只写“做成卡片/流程图”。
- 每页必须写“模块间关系”“主视觉与辅助内容面积预算”“生成风险提示”。
- 主视觉占主体区建议 >= 55%；流程路径页流程主视觉 >= 60%；辅助模块 <= 30%。
- 贯穿型能力如监测中心、审核中心、安全中心，应作为底座、背景条、侧挂治理层或护栏，不得塞进流程中间。
- 页面文案只写真正上屏的短文案；长解释放到主视觉内容或演讲备注。
- 输出后附自检：内容保留、主视觉明确、模块关系清楚、面积预算合理、区域词未上屏、未绑定设计语言。
```

## B2.5 生成就绪导演稿 Prompt

```text
你是产品解决方案专家 + PPT 视觉专家 + PPT 生成工程师。
请基于“页面描述_优化版”和当前设计语言资料，生成“生成就绪导演稿”。

【页面描述_优化版】
粘贴或引用路径

【设计语言资料】
- design.md：
- style-card.md：
- slide-type-map.md：
- contact-sheet：
- fixed assets：

【输出目标】
- 首轮输出格式：HTML+CSS 预览
- 画布尺寸：1280 × 720 px，16:9
- 后续输出：PPTX，并需要 PPTX 实际渲染截图与 PNG 对比

【输出要求】
1. 明确输出格式，不要只输出文字解释。
2. 给出画布参数：标题区、主体区、底部区、安全边距。
3. 从 design.md 读取并列出色彩、字体、字号、字重、母版元素、组件样式、线条/箭头样式和固定资产路径。
4. 为每页给出关键区域坐标或比例，例如主视觉区域、侧挂模块、底部价值区。
5. 给出组件映射：内容节点 -> 视觉组件 -> 坐标/尺寸 -> 样式 token。
6. 给出字体字号参数：大标题、小标题、内容文字、页脚/来源的字号档位和字重；digital-zhejiang 应使用微软雅黑，禁止微软雅黑 Light、宋体、特殊字体。
7. 给出固定资产引用：例如 digital-zhejiang 的 `city-skyline.png` 必须作为右下城市剪影原图引用，禁止自行生成或重绘。
8. 给出连接规则：箭头起点、终点、方向、禁止连接。
9. 给出生成优先级：如果约束冲突，主视觉面积和连接关系正确优先于局部美观；中文清晰可读优先于塞满内容。
10. 给出负面约束：不要双主视觉、不要区域词上屏、不要普通卡片堆叠、不要低于设计系统字号档位、不要重绘固定母版资产。
11. 给出 HTML/PPTX 审查门禁，包含字体替换、换行、溢出、字号越界和固定资产是否引用原图的检查。

注意：
- 页面描述_优化版仍然是内容结构层，不改业务事实。
- 生成就绪导演稿是生成执行层，可以写来自设计系统的颜色、字体、母版规则和固定资产路径。
- 所有坐标、区域词、组件名称都是执行指令，不是上屏文案。
```

## D. HTML 设计预览 Prompt

```text
你是一个 PPT 设计预览专家。请基于以下页面结构文档和设计语言规范，先生成 HTML 设计预览，不要直接生成 PPTX。

【页面结构文档】
粘贴标准交付文档或页面结构导演稿。该文档只描述页面内容、结构、排布和表达重点，不是视觉风格来源。

【生成就绪导演稿】
粘贴或引用路径。该文档已经读取设计语言，包含输出格式、画布参数、坐标/比例、样式 token、组件映射、连接规则和生成优先级。

【设计语言规范】
粘贴 design.md 摘要或路径

【风格摘要】
粘贴 style-card 和 slide-type-map 摘要

【截图样例】
粘贴 contact-sheet.png 路径或说明。它只用于风格靠齐，不要求逐页复刻。

【固定母版资产】
粘贴 registry/design.md 中的固定资产路径，例如 city-skyline.png。固定资产必须引用原图，不得生成或重绘。

【输出要求】
1. 输出可打开的 HTML 预览，画布固定 16:9，例如 1280x720
2. 每页严格对照页面结构文档，不要擅自改标题、模块顺序或表达重点
3. 每页优先遵循“页面结构导演稿”，包括版式、主视觉、侧边模块、底部标签和出图禁忌
4. 再根据生成就绪导演稿和 design.md 把页面结构翻译成具体视觉风格
5. 每页必须符合当前设计语言；版式不必和截图完全一致，但颜色、字体、母版固定元素、固定资产、组件语法和信息密度必须一致
6. 能靠齐样例页型时尽量靠齐；内容需要新结构时按 design.md 做变体
7. 页面结构中的区域词只用于排版定位；不得把“左侧：”“右侧：”“顶部：”“底部：”“中部：”“侧边：”显示成模块标题
8. 每页输出截图，并生成 contact sheet
9. 数据图表页要准确呈现数据
10. 图片页使用提供路径或占位符路径
11. 母版固定资产必须使用提供路径原图，例如 city-skyline.png，不能用 CSS/SVG/AI 重新生成
12. HTML 结构要尽量可映射到 PPTX：文本、矩形、流程、矩阵、箭头、图片框分层清楚
13. 如果生成就绪导演稿中已有坐标/比例和组件映射，优先执行；只有在明显冲突时才调整，并在审查报告中说明

【最终布局审查要求】
生成 HTML 后，必须读取并执行 final-layout-review.md：
- 用产品解决方案专家视角审查：业务价值、场景闭环、方案边界、可信依据、管理抓手
- 用 PPT 专家视角审查：视觉重心、页面饱满度、结构清晰度、信息密度、主视觉表达、对齐间距、底部收束
- 用微观布局视角审查：区域均匀铺设、图文高度一致、同组间距、内边距、基线、连接线、页脚安全区和品牌角标避让
- 检查区域词是否误上屏：左侧/右侧/顶部/底部/中部/侧边等前缀不得作为可见文案
- 用设计语言视角审查：是否符合 design.md、style-card、slide-type-map 和样例气质
- 输出 100 分评分、P0/P1/P2 修改清单、是否通过
- 低于 80 分或存在 P0 问题时，不得进入 PPTX 映射，只能提出修正计划
```

## E. PPTX 映射生成 Prompt

```text
你是一个 PPTX 映射专家。请把已经人工确认、且通过最终布局审查的 HTML 设计预览映射为可编辑 PPTX。

【已确认 HTML 预览】
粘贴 index.html / contact sheet / 关键截图路径。

【页面结构文档】
粘贴标准交付文档或页面结构导演稿。

【设计语言规范】
粘贴 design.md / style-card / slide-type-map 摘要或路径。

【映射要求】
1. 不要默认把整页 HTML 截图贴进 PPT。
2. 标题、正文、模块文字、基础形状、流程箭头、矩阵、分层架构尽量映射成 PPT 原生可编辑对象。
3. 复杂背景、局部复杂插画或产品截图可作为图片插入，但要说明哪些元素被栅格化。
4. 输出 PPTX 后检查页数、可打开性、关键文本可编辑性、关键页面与 HTML 的一致性。
5. PPTX 必须导出实际渲染截图，并与 HTML/PNG 预览并排对比。
6. 如果原生可编辑 PPTX 的实际渲染与 HTML/PNG 明显不一致，先修复映射；若短期无法修复且用户优先要视觉还原，输出“视觉一致版 PPTX”和“可编辑原生版 PPTX”两个文件并说明取舍。

【最终布局审查要求】
PPTX 生成后必须再次执行 final-layout-review.md：
- 使用 PPTX 导出的截图或实际预览图，不只看代码
- 对比 HTML/PNG 预览与 PPTX 实际渲染图，检查字体换行、模块位置、箭头方向、连接关系、主辅面积、母版元素是否漂移
- 同时给出产品解决方案专家意见和 PPT 专家意见
- 总分 >= 80 且无 P0 问题，才允许输出为最终 PPTX
- 不通过时，回到 HTML 或 PPTX 映射代码修改，不得宣称完成
```

## G. 最终布局审查 Prompt

```text
请按 final-layout-review.md 对以下 HTML/PPTX 预览做最终门禁审查。

【审查对象】
- 文件路径：
- 截图/contact sheet 路径：
- 若审查对象为 PPTX，必须提供 PPTX 实际渲染截图：
- HTML/PNG 与 PPTX 渲染对比图：
- 当前设计语言：
- 受众与场景：

【审查角色】
1. 产品解决方案专家：检查业务价值、场景闭环、方案边界、可信依据、管理抓手。
2. PPT 专家：检查视觉重心、页面饱满度、结构清晰度、信息密度、主视觉表达、对齐间距、底部收束。
```
