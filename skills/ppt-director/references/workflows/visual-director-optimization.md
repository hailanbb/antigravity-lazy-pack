# Visual Director Optimization Workflow

本工作流用于在“内容型 PPT MD”和“HTML 设计预览”之间增加一层页面视觉导演优化。

核心目的：不是改业务内容，而是把每页从“内容说明”升级为“可生成、可审查、可映射的页面分镜”。

注意：本阶段分两步。

1. **页面描述优化**：输出不绑定设计语言的 `页面描述_优化版`。
2. **生成就绪导演稿**：读取当前 `design.md` 后，补齐输出格式、画布参数、坐标区间、样式 token、组件映射、连接规则和生成优先级。

## 使用时机

当用户已有逐页 PPT 内容、页面结构导演稿或素材型 MD，并准备生成 HTML/PPTX 时，必须先执行本阶段。

尤其适用于：

- 内容很多，容易生成文字堆砌页。
- 页面已经有标题和要点，但主视觉不够明确。
- 需要遵循某个设计语言，但内容结构层不应写具体视觉风格。
- 用户指出 HTML 像普通卡片页、没有按设计语言表达、结构不够像 PPT。
- 用户或评审指出页面描述缺少输出格式、坐标、尺寸、色彩/字号/组件参数、生成优先级。

## 输入

- 内容型 PPT MD 或标准交付文档。
- 原始素材约束和必须保留内容。
- 受众/评审专家卡片（如有）。
- 页面结构导演稿格式。
- 当前设计语言资料（仅用于第二步生成执行层）：`design.md`、`style-card.md`、`slide-type-map.md`、`contact-sheet.png`。
- 当前设计语言固定资产（仅用于第二步生成执行层）：例如 digital-zhejiang 的 `city-skyline.png`。

注意：第一步 `页面描述_优化版` 不读取 `design.md` 来写颜色、字体、母版元素或截图页型；第二步 `生成就绪导演稿` 必须读取 `design.md` 和 `registry.yml` 资产路径，把设计系统转成执行参数。

## 输出

生成三类产物，至少要有“视觉导演优化版 MD”：

```text
[项目名]_原始页面描述.md        # 可选，保存用户输入或整理后的原始版本
[项目名]_页面描述_优化版.md     # 必需，作为 HTML/PPTX 生成的唯一内容基准
[项目名]_描述优化说明.md        # 推荐，说明保留、重构和风险处理
[项目名]_生成就绪导演稿.md      # 必需，作为代码 Agent 的直接输入
```

每页仍然使用 `page-structure-brief-schema.md` 的标准字段：

- 页面标题
- 本页核心结论
- 页面类型
- 页面版式
- 主视觉设计
- 主视觉内容
- 模块间关系
- 主视觉与辅助内容面积预算
- 页面文案
- 页面想表达的管理价值
- 页面底部总结语
- 生成风险提示
- 出图要求

## 优化原则

1. **保留业务事实**：不得删掉用户指定必须保留的政策、链路、产品能力、场景、指标、边界。
2. **压缩上屏表达**：模块标题尽量 4-12 字，模块说明尽量 12-30 字。
3. **强化主视觉**：每页必须说明主视觉承担什么逻辑证明，而不是只说“卡片/流程图/架构图”。
4. **分离内容与风格**：不得写具体设计语言、颜色、字体、母版元素、品牌角标、截图样例。
5. **清洗区域词风险**：`左侧/右侧/顶部/底部/中部/侧边` 只能出现在版式说明中，不得作为上屏模块标题。
6. **控制信息密度**：长解释放入主视觉内容或演讲备注，上屏文案只保留短句。
7. **提升可映射性**：结构应能被 HTML/PPTX 映射成文本、矩形、流程、矩阵、分层架构、箭头和图片框。
8. **显性化模块关系**：页面多个模块之间必须写清并列、支撑/底座、总分、贯穿型能力、侧挂治理层等关系。
9. **声明面积主次**：每页必须有 1 个主视觉。主视觉占主体区建议 >= 55%；流程路径页应 >= 60%；辅助模块 <= 30%。
10. **识别双主视觉冲突**：如果两个视觉板块面积接近，必须在优化版中重新分配主次或建议拆页。
11. **处理贯穿型能力**：监测中心、审核中心、安全中心、运营中心等贯穿型能力，应作为底座、背景条、侧挂治理层或轻量护栏，不要塞进流程中间。
12. **执行参数后置**：输出格式、画布参数、坐标、色彩、字号、组件样式、箭头样式和生成优先级，统一放到 `生成就绪导演稿`。

## 执行步骤

1. 读取内容型 MD，列出全局必须保留内容。
2. 逐页判断页面类型：分层架构、业务流程、能力体系、对比分析、场景价值、实施路径、案例成效、机制说明。
3. 逐页明确“主视觉证明对象”：这页观众主要看什么图才能理解结论。
4. 把原始素材拆为：
   - 主视觉节点
   - 上屏短文案
   - 管理价值
   - 备注/解释性材料
5. 逐页补充模块间关系：
   - 并列关系 -> 等宽等高排列
   - 支撑/底座关系 -> 置于底部并横跨全宽
   - 总分关系 -> 主模块面积至少为从属模块 2 倍
   - 贯穿型能力 -> 底座、背景条、侧挂治理层或轻量护栏
6. 逐页补充面积预算：
   - 主视觉占主体区比例
   - 辅助模块占比
   - 价值总结占比
7. 检查每页是否只服务一个核心结论。
8. 检查是否存在双主视觉冲突、模块无关系堆叠、价值点抢主视觉、贯穿型能力位置错误。
9. 清洗可见文案中的区域词前缀。
10. 输出视觉导演优化版 MD 和描述优化说明。
11. 读取设计语言资料，生成设计语言对表。
12. 合成生成就绪导演稿：
   - 明确输出格式：HTML+CSS、SVG、python-pptx 或 HTML+PPTX。
   - 明确画布尺寸、安全区、标题区、主体区、底部区。
   - 从设计系统提取颜色、字体、母版元素、组件形态、线条/箭头样式。
   - 从 registry 或 design.md 提取固定资产路径，例如城市剪影 `city-skyline.png`。
   - 给出关键区域坐标或比例。
   - 给出主视觉、辅助模块和底部价值区的面积预算。
   - 给出箭头起点、终点、方向和禁止连接。
   - 给出生成优先级。
13. 再进入 HTML 预览生成。

## 质量检查

输出前检查：

- 是否保留了用户明确要求的核心内容？
- 是否每页都有明确页面类型？
- 是否每页都有主视觉设计和主视觉内容？
- 是否区分了上屏文案与解释性背景？
- 是否没有写入设计语言、颜色、字体和截图样例？
- 是否没有把 `左侧：/右侧：/侧边：` 作为可见标题？
- 是否写清模块间关系，而不是多个模块无关系堆叠？
- 是否声明主视觉与辅助内容面积预算？
- 是否避免双主视觉冲突？
- 若是流程路径页，流程是否为主视觉且占主体区 60% 以上？
- 价值点是否不超过 3 条，且作为底部收束或侧边点缀？
- 贯穿型模块是否没有被塞进流程中间？
- 是否能直接交给 HTML 生成阶段读取？
- 是否已经生成第二层 `生成就绪导演稿`？
- 生成就绪导演稿是否明确输出格式、画布参数、坐标区间、设计系统样式参数和固定资产路径？
- 生成就绪导演稿是否给出约束冲突时的优先级？

## 推荐 Prompt

```text
你是产品解决方案专家 + PPT 视觉专家。请把以下内容型 PPT MD 二次优化为“视觉导演优化版 MD”。

目标：
1. 不改业务事实，不丢关键素材。
2. 把每页改写为可生成 HTML/PPTX 的页面结构分镜。
3. 强化每页主视觉、页面类型、版式骨架、上屏短文案和管理价值。
4. 不写具体设计语言、颜色、字体、母版元素或截图样例。
5. 页面版式中的左侧/右侧/顶部/底部/中部/侧边只作为排版指令，不得作为可见模块标题。
6. 显性化模块间关系，并声明主视觉与辅助内容面积预算。
7. 检查双主视觉冲突、模块无关系堆叠、贯穿型能力位置错误、箭头关系不清等生成风险。

输出：
- 每页按 page-structure-brief-schema.md 输出。
- 模块标题尽量短，说明文字克制。
- 主视觉内容必须保留核心素材。
- 生成后给出描述优化说明和自检结果。
```

## 生成就绪导演稿 Prompt

```text
请读取当前页面描述_优化版和设计语言资料，生成“生成就绪导演稿”。

【页面描述_优化版】
粘贴或引用路径

【设计语言资料】
- design.md：
- style-card.md：
- slide-type-map.md：
- contact-sheet：
- fixed assets：

【输出目标】
HTML+CSS 预览，画布 1280×720；后续映射 PPTX。

请输出：
1. 生成目标与输出格式
2. 画布参数和安全区
3. 来自 design.md 的色彩、字体、母版元素、组件样式、箭头样式和固定资产路径
4. 每页关键区域坐标或比例
5. 组件映射：内容节点 -> 视觉组件 -> 坐标/尺寸 -> 样式 token
6. 箭头和连接规则：起点、终点、方向、禁止连接
7. 生成优先级：主视觉面积和关系正确优先于局部美观
8. 负面约束和审查门禁

注意：
- 不要把坐标和区域词当作上屏文案。
- 样式参数必须来自当前设计系统，不得凭空编造。
- 固定资产必须引用原文件，例如 city-skyline.png，不得自行生成或重绘。
- 如果设计系统与页面描述有冲突，以页面核心结论和主视觉关系为先，再做设计变体。
```
